Historial de Windows

Sistema operativo de disco

Las primeras computadoras no tenían dispositivos de almacenamiento modernos como discos duros, unidades ópticas o almacenamiento flash. Los primeros métodos de almacenamiento usaban tarjetas perforadas, cintas de papel, cinta magnética e incluso casetes de audio.
El almacenamientoen disquete y disco duro requiere software para leer, escribir y administrar los datos que almacenan. El sistema operativo de disco (DOS) es un sistema operativo que usa la computadora para permitir que estos dispositivos de almacenamiento de datos lean y escriban archivos. DOS proporciona un sistema de archivos que organiza los archivos de una manera específica en el disco. Microsoft compró DOS y desarrolló MS-DOS.
MS-DOS usó una línea de comandos como interfaz para que las personas crearan programas y manipularan archivos de datos, como se muestra en la salida del comando. Los comandos de DOS se muestran en negrita.

Con MS-DOS, la computadora tenía un conocimiento básico de cómo acceder a la unidad de disco y cargar los archivos del sistema operativo directamente desde el disco como parte del proceso de arranque. Cuando se cargó, MS-DOS podía acceder fácilmente al disco porque estaba integrado en el sistema operativo.
Las primeras versiones de Windows consistían en una interfaz gráfica de usuario (GUI) que se ejecutaba sobre MS-DOS, comenzando con Windows 1.0 en 1985. El sistema operativo de disco aún controlaba la computadora y su hardware. Un sistema operativo moderno como Windows 10 no se considera un sistema operativo de disco. Está construido sobre Windows NT, que significa "Nuevas Tecnologías". El propio sistema operativo tiene el control directo de la computadora y su hardware. NT es un sistema operativo con soporte para múltiples procesos de usuario. Esto es muy diferente al MS-DOS de un solo proceso y un solo usuario.
Hoy en día, muchas cosas que solían lograrse a través de la interfaz de línea de comandos de MS-DOS se pueden lograr en la GUI de Windows. Aún puede experimentar cómo era usar MS-DOS abriendo una ventana de comandos, pero lo que ve ya no es MS-DOS, es una función de Windows. Para experimentar un poco cómo era trabajar en MS-DOS, abra una ventana de comandos escribiendo cmd en la búsqueda de Windows y presionando Enter . La tabla enumera algunos comandos que puede utilizar. Ingrese ayuda seguida del comando para obtener más información sobre el comando.

Versiones de Windows

Desde 1993, ha habido más de 20 versiones de Windows que se basan en el sistema operativo NT. La mayoría de estas versiones estaban destinadas al público en general y a las empresas debido a la seguridad de archivos que ofrece el sistema de archivos que utilizaba el sistema operativo NT. Las empresas también adoptaron sistemas operativos Windows basados ​​en NT OS. Esto se debe a que muchas ediciones se crearon específicamente para estaciones de trabajo, profesionales, servidores, servidores avanzados y servidores de centros de datos, por nombrar solo algunas de las muchas versiones diseñadas específicamente.
A partir de Windows XP, estaba disponible una edición de 64 bits. El sistema operativo de 64 bits era una arquitectura completamente nueva. Tenía un espacio de direcciones de 64 bits en lugar de un espacio de direcciones de 32 bits. Esto no es simplemente el doble de espacio porque estos bits son números binarios. Mientras que Windows de 32 bits puede abordar un poco menos de 4 GB de RAM, Windows de 64 bits teóricamente puede abordar 16.8 millones de terabytes. Cuando el sistema operativo y el hardware admiten el funcionamiento de 64 bits, se pueden utilizar conjuntos de datos extremadamente grandes. Estos grandes conjuntos de datos incluyen bases de datos muy grandes, computación científica y manipulación de video digital de alta definición con efectos especiales. En general, las computadoras y los sistemas operativos de 64 bits son compatibles con programas anteriores de 32 bits, pero los programas de 64 bits no se pueden ejecutar en hardware antiguo de 32 bits.
Con cada lanzamiento posterior de Windows, el sistema operativo se ha vuelto más refinado al incorporar más funciones. Windows 7 se ofreció con seis ediciones diferentes, Windows 8 con hasta cinco y Windows 10 con ocho ediciones diferentes. Cada edición no solo ofrece diferentes capacidades, sino también diferentes puntos de precio. Microsoft ha dicho que Windows 10 es la última versión de Windows y que Windows se ha convertido en un servicio en lugar de solo un sistema operativo. Dicen que en lugar de comprar nuevos sistemas operativos, los usuarios simplemente actualizarán Windows 10.

GUI de Windows

Windows tiene una interfaz gráfica de usuario (GUI) para que los usuario trabajen con archivos de datos y software. La GUI tiene un área principal llamada Escritorio, que se muestra a continuación.

El escritorio se puede personalizar con varios colores e imágenes de fondo. Windows admite varios usuarios, por lo que cada usuario puede personalizar el escritorio a su gusto. El escritorio puede almacenar archivos, carpetas, accesos directos a ubicaciones, programas y aplicaciones. El escritorio también tiene un icono de papelera de reciclaje, donde los archivos se almacenan cuando el usuario los elimina. Los archivos se pueden restaurar desde la papelera de reciclaje o se pueden vaciar los archivos de la papelera de reciclaje, lo que realmente los elimina.
En la parte inferior del escritorio está la barra de tareas. La barra de tareas tiene tres áreas que se utilizan para diferentes propósitos. A la izquierda está el menú Inicio. Se utiliza para acceder a todos los programas instalados, opciones de configuración y la función de búsqueda. En el centro de la barra de tareas, los usuarios colocan iconos de inicio rápido que ejecutan programas específicos o abren carpetas específicas cuando se hace clic en ellos. Finalmente, a la derecha de la barra de tareas está el área de notificación. El área de notificación muestra, de un vistazo, la funcionalidad de muchos programas y funciones diferentes. Por ejemplo, un icono de sobre parpadeante puede indicar un nuevo correo electrónico, o un icono de red con una "x" roja puede indicar un problema con la red.
A menudo, al hacer clic con el botón derecho en un icono, aparecerán funciones adicionales que se pueden utilizar. Esta lista se conoce como menú contextual, que se muestra en la figura.
Hay menús contextuales para los íconos en el área de notificación, para los íconos de inicio rápido, íconos de configuración del sistema y para archivos y carpetas. El menú contextual proporciona muchas de las funciones más utilizadas con solo hacer clic. Por ejemplo, el menú contextual de un archivo contendrá elementos como copiar, eliminar, compartir e imprimir. Para abrir carpetas y manipular archivos, Windows usa el Explorador de archivos de Windows.

Vulnerabilidades del sistema operativo

Los sistemas operativos constan de millones de líneas de código. El software instalado también puede contener millones de líneas de código. Contodo este código vienen las vulnerabilidades. Una vulnerabilidad es un defecto o debilidad que un atacane puede aprovechar para reducir la viabilidad de la información de una computadora Para aprovechar una vulnerabilidad del sistema operativo, el atacante debe usar una técnica o una herramienta para aprovechar la vulnerabilidad. El atacante puede entonces usar la vulnerabilidad para hacer que la computadora actúe de una manera diferente a su diseño previsto. En general, el objetivo es obtener el control no autorizado de la computadora, cambiar los permisos o manipular o robar datos.

Capa de abstracción de hardware

Las computadoras con Windows usan muchos tipos diferentes de hardware. El sistema operativo puede instalarse en una computadora comprada o en un computador ensamblado por el usuario. Cuando se instala el sistema operativo, debe aislarse de las diferencias de hardware. La arquitectura básica de Windows se muestra en la figura.

Una capa de abstracción de hardware (HAL) es un software que maneja toda la comunicación entre el hardware y el kernel. El kernel es el núcleo del sistemap operativo y tiene control sobre toda la computadora. Maneja todas las solicitudes de entrada y salida, la memoria y todos los periféricos conectados a la PC. En algunos casos, el kernel aún se comunica con el hardware directamente, por lo que no es completamente independiente de HAL. HAL también necesita el kernel para realizar algunas funciones.

Modo de usuario y modo de kernel

Como se muestra en la figura siguiente, hay dos modos diferentes en los que una CPU funciona cuando la computadora tiene Windows instalado: el modo usuario y el modo kernel.

Las aplicaciones instaladas se ejecutan en modo de usuario y el código del sistema operativo se ejecuta en modo de kernel. El código que se ejecuta en modo kernel tiene acceso sin restricciones al hardware subyace y es capaz de ejecutar cualquier instrucción de la CPU. El código del modo kernel también puede hacer referencia a cualquier dirección de memoria directamente.
Generalmente reservado para las funciones más confiables del SO, los bloques en el código que se ejecuta en modo kernel detienen el funcionamiento de toda la computadora. Por el contrario, los programas, como las aplicaciones de usuario, se ejecutan en modo de usuario y no tienen acceso directo a ubicaciones de hardware o memoria. El código de modo de usuario debe pasar por el sistema operativo para acceder a los recursos de hardware. Debido al aislamiento proporcionado por el modo de usuario, los bloques en el modo usuario están restringidos solo a la aplicación y son recuperables. La mayoría de los programas de Windows se ejecutan en modo de usuario, Controladores de dispositivo.
Todo el código que se ejecuta en modo kernel usa el mismo espacio de direcciones. Los controladores en modo kernel no están aislados del sistema operativo. Si ocurre un error con el controlador ejecutándose en modo kernel y escribe en el espacio de direcciones incorrecto, el SO u otro controlador en modo kernel podría verse afectados negativamente. En este sentido, el controlador podría bloquearse, provocando que todo el sistema operativo se bloquee.
Cuando se ejecuta el código del modo de usuario, el kernel le otorga su propio espacio de direcciones restringido, junto con un proceso creado específicamente para la aplicación. El motivo de esta funcionalidad ees principalmente patra evitar que las aplicacionescambien el código del sistema operativo que se ejecuta al mismo tiempo. Al tener su propio proceso, esa aplicación tiene su propio espacio de direcciones privadas, lo que hace que otras aplicaciones no puedan modificar los datos que contiene. Esto también ayuda a evitar que el SO y otras aplicaciones se bloqueen si esa aplicación falla.

Antes de poder utulizar un dispositivo de almacenamiento, como un disco, debe formatearse con un sstema de archivos. A su vez, antes de que un sistema de archivos pueda instalarse en un dispositivo de almacenamiento, es necesario particionar el dispositivo. Un disco duro se divide en áreas llamadas particiones. Cada partición es una unidad de almacenamiento lógica que se puede formatear para almacenar información, como archivos de datos o aplicaciones. Durante el proceso de instalación, la mayoría de los sistemas operativos particionan y formatean automáticamente el espacio disponible en el disco con un sistema de archivos como NTFS.
El formateo NTFS crea estructuras importantes en el disco para el almacenamiento de archivos y tablas para registrar las ubicaciones de los archivos:
  • Sector de arranque de partición. Estos son los primeros 16 sectores de la unidad. Contiene la ubicación de la tabla maestra de archivos (MFT). Los últimos 16 sectores contienen una copia del sector de arranque.
  • Tabla maestra de archivos (MFT). Esta tabla contiene las ubicaciones de todos los archivos y directorios de la partición, incluidos los atributos de archivo, como información de seguridad y marcas de tiempo.
  • Archivo de sistema. Estos son archivos ocultos que almacenan información sobre otros volúmenes y atributos de archivo.
  • Área de archivos. El área principal de la partición donde se almacenan los archivos y directorios.
Nota: Al formatear una partición, es posible que los datos anteriores aún se puedan recuperar porque no todos los datos se eliminan por completo. Se puede examinar el espacio libre y se pueden recuperar archivos que pueden comprometer la seguridad. Se recomienda realizar una limpieza segura en una unidad que se está reutilizando. El borrado seguro escribirá datos en todo el disco varias veces para garantizar que no queden datos.

Flujos de datos alternativos

NTFS almacena archivos como una serie de atributos, como el nombre del archivo o una marca de tiempo. Los datos que contiene el archivo se almacenan en el atributo $DATA y se conocen como flujo de datos. Al usar NTFS, puede conectar flujos de datos alternativos (ADS) al archivo. A veces, esto lo usan aplicaciones que almacenan información adicional sobre el archivo. El ADS es un factor importante cuando se habla de malware. Esto se debe a que es fácil ocultar datos en un ADS. Un atacante podría almacenar código malicioso dentro de un ADS que luego se puede llamar desde un archivo diferente.
En el sistema de archivos NTFS, un archivo con ADS se identifica después del nombre de archivo y dos puntos, por ejemplo, Testfile.txt: ADS . Este nombre de archivo indica que un ADS llamado ADS está asociado con el archivo llamado Testfile.txt . Un ejemplo de ADS se muestra en la salida del comando.

Proceso de arranque de Windows

Muchas acciones ocurren entre el momento en que se presiona el botón de encendido de la computadora y Windows está completamente cargado, como se muestra en la figura.

Esto se conoce como el proceso de inicio de Windows.
Existen dos tipos de wirmware de computadora:
  • Sistema básico de entrada y salida (BIOS). El firmware de la BIOS se creó a principios de la década de 1980 y funciona de la misma manera que cuando se creó. A medida que evolucionaron las computadoras, se hizo difícil para el firmware del BIOS admitir todas las nuevas funciones solicitadas por los usuarios.
  • Interfaz de firmware extensible unificada (UEFI). UEFI se diseño para reemplazar BIOS y admitir las nuevas funciones.
En el firmware del BIOS, el proceso comienza con la fase de inicialización del BIOS. Esto es cuando se inicializan los dispositivos de hardware y se realiza una autoprueba de encendido (POST) para asegurarse de que todos estos dispositivos se estén comunicando. Cuando se descubre el disco del sistema, la POST finaliza. La última instrucción en la POST es buscar el registro de arranque maestro (MBR). El MBR contiene un pequeño programa que se encarga de localizar y cargar el sistema operativo. El BIOS ejecuta este código y el sistema operativo comienza a cargarse. A diferencia del firmware del BIOS, el firmware UEFI tiene mucha visibilidad del proceso de arranque. UEFI arranca cargando archivos de programa EFI, almacenados como archivos .efi en una partición de disco especial, conocida como EFI System Partition (ESP). Nota : una computadora que usa UEFI almacena el código de arranque en el firmware. Esto ayuda a aumentar la seguridad de la computadora en el momento del arranque porque la computadora pasa directamente al modo protegido. Ya sea que el firmware sea BIOS o UEFI, después de encontrar una instalación válida de Windows, se ejecuta el archivo Bootmgr.exe . Bootmgr.exe cambia el sistema del modo real al modo protegido para que se pueda utilizar toda la memoria del sistema. Bootmgr.exe lee la base de datos de configuración de arranque (BCD). El BCD contiene cualquier código adicional necesario para iniciar la computadora, junto con una indicación de si la computadora está saliendo de la hibernación o si se trata de un inicio en frío. Si el equipo sale de la hibernación, el proceso de arranque continúa con Winresume.exe . Esto permite que la computadora lea el archivo Hiberfil.sys que contiene el estado de la computadora cuando se puso en hibernación. Si la computadora se inicia desde un inicio en frío, entonces se carga el archivo Winload.exe . El archivo Winload.exe crea un registro de la configuración del hardware en el registro. El registro es un registro de todas las configuraciones, opciones, hardware y software que tiene la computadora. El registro se explorará en profundidad más adelante en este capítulo. Winload.exe también utiliza la firma de código en modo kernel (KMCS) para asegurarse de que todos los controladores estén firmados digitalmente. Esto asegura que los controladores sean seguros para cargar cuando se inicia la computadora. Una vez examinados los controladores, Winload.exe ejecuta Ntoskrnl.exe, que inicia el kernel de Windows y configura HAL. Finalmente, el Subsistema de administrador de sesión (SMSS) lee el registro para crear el entorno de usuario, iniciar el servicio Winlogon y preparar el escritorio de cada usuario a medida que inician sesión.

Apagado de Windows

Siempre es mejor realizar un apagado adecuado para apagar la computadora. Los archivos que quedan abiertos, los servicios que se cierran fuera de servicio y las aplicaciones que se cuelgan pueden dañarse si se apaga la alimentación sin informar primero al sistema operativo. La computadora necesita tiempo para cerrar cada aplicación, apagar cada servicio y registrar cualquier cambio de configuración antes de que se pierda la energía.
Durante el apagado, la computadora cerrará primero las aplicaciones en modo de usuario, seguidas de los procesos en modo kernel. Si un proceso de modo de usuario no responde dentro de un cierto período de tiempo, el sistema operativo mostrará una notificación y permitirá al usuario esperar a que la aplicación responda o finalizar el proceso por la fuerza. Si un proceso en modo kernel no responde, parecerá que el apagado se cuelga y puede que sea necesario apagar la computadora con el botón de encendido.
Hay varias formas de apagar una computadora con Windows: las opciones de energía del menú Inicio, el comando de línea de comando apagar y usar Ctrl + Alt + Supr y hacer clic en el ícono de energía. Hay tres opciones diferentes entre las que elegir al apagar la computadora:

  • Apagar. Apaga el equipo.
  • Reiniciar. Reinicia la computadora (apague y encienda).
