¿Qué es Linux?

Linux es un sistema operativo que se creó en 1991. Linux es de código abierto, rápido, confiable y pequeño. Requiere muy pocos recursos de hardware para ejecutarse y es altamente personalizable. A diferencia de otros sistemas operativos como Windows y Mac OS X, Linux fue creado y actualmente es mantenido por una comunidad de programadores. Linux es parte de varias plataformas y se puede encontrar en dispositivos en cualquier lugar, desde "relojes de pulsera hasta supercomputadoras".
Otro aspecto importante de Linux es que está diseñado para conectarse a la red, lo que simplifica mucho la escritura y el uso de aplicaciones basadas en red. Debido a que Linux es de código abierto, cualquier persona o empresa puede obtener el código fuente del kernel, inspeccionarlo, modificarlo y volver a compilarlo a voluntad. También se les permite redistribuir el programa con o sin cargos.
Una distribución de Linux es el término que se utiliza para describir paquetes creados por diferentes organizaciones. Las distribuciones (o distribuciones) de Linux incluyen el kernel de Linux con herramientas y paquetes de software personalizados. Si bien algunas de estas organizaciones pueden cobrar por su soporte de distribución de Linux (orientado a empresas basadas en Linux), la mayoría de ellas también ofrecen su distribución de forma gratuita sin soporte. Debian, Red Hat, Ubuntu, CentOS y SUSE son solo algunos ejemplos de distribuciones de Linux.

El valor de Linux

Linux es a menudo el sistema operativo de elección en el Centro de operaciones de seguridad (SOC). Estas son algunas de las razones para elegir Linux:

  • Linux es de código abierto : cualquier persona puede adquirir Linux sin cargo y modificarlo para que se ajuste a sus necesidades específicas. Esta flexibilidad permite a los analistas y administradores crear un sistema operativo a medida específicamente para el análisis de seguridad.
  • La CLI de Linux es muy poderosa : si bien una GUI facilita la realización de muchas tareas, agrega complejidad y requiere más recursos informáticos para ejecutarse. La interfaz de línea de comandos de Linux (CLI) es extremadamente poderosa y permite a los analistas realizar tareas no solo directamente en una terminal, sino también de forma remota.El usuario tiene más control sobre el sistema operativo : el usuario administrador en Linux, conocido como usuario root o superusuario, tiene poder absoluto sobre la computadora. A diferencia de otros sistemas operativos, el usuario root puede modificar cualquier aspecto de la computadora con unas pocas teclas. Esta capacidad es especialmente valiosa cuando se trabaja con funciones de bajo nivel, como la pila de red. Permite al usuario root tener un control preciso sobre la forma en que el sistema operativo maneja los paquetes de red.
  • Permite un mejor control de la comunicación de la red : el control es una parte inherente de Linux. Debido a que el sistema operativo se puede ajustar en prácticamente todos los aspectos, es una gran plataforma para crear aplicaciones de red. Esta es la misma razón por la que muchas excelentes herramientas de software basadas en red están disponibles solo para Linux.

Linux en el SOC

La flexibilidad proporcionada por Linux es una gran característica para el SOC. Todo el sistema operativo se puede adaptar para convertirse en la plataforma de análisis de seguridad perfecta. Por ejemplo, los administradores pueden agregar solo los paquetes necesarios al sistema operativo, haciéndolo ágil y eficiente. Se pueden instalar y configurar herramientas de software específicas para que funcionen en conjunto, lo que permite a los administradores crear una computadora personalizada que se adapte perfectamente al flujo de trabajo de un SOC.

Herramientas Linux

Además de las herramientas específicas de SOC, las PC Linux que se usan en el SOC a menudo contienen herramientas de prueba de penetración. También conocida como PenTesting, una prueba de penetración es el proceso de buscar vulnerabilidades en una red o computadora atacándola. Los generadores de paquetes, los escáneres de puertos y los exploits de prueba de concepto son ejemplos de herramientas de PenTesting.
Kali Linuz es una distribución de Linux que agrupa muchas herramientas de penetración en una única distribución de Linux.

El Shell de Linux

En Linux, el usuario se comunica con el sistema operativo mediante la CLI o la GUI. Linux a menudo se inicia en la GUI de forma predeterminada. Esto oculta la CLI al usuario. Una forma de acceder a la CLI desde la GUI es a través de una aplicación de emulador de terminal. Estas aplicaciones proporcionan acceso de usuario a la CLI y, a menudo, se denominan como una variación de la palabra "terminal". En Linux, los emuladores de terminal más populares son Terminator, eterm, xterm, konsole y gnome-terminal.

Comandos básicos

Los comandos de Linux son programas creados para realizar una tarea específica. Utilice el comando man (abreviatura de manual) para obtener documentación sobre los comandos. Como ejemplo, man ls proporciona documentación sobre el comando ls del manual del usuario.
Debido a que los comandos son programas almacenados en el disco, cuando un usuario escribe un comando, el shell debe encontrarlo en el disco antes de que pueda ejecutarse. El shell buscará comandos escritos por el usuario en directorios específicos e intentará ejecutarlos. La lista de directorios comprobados por el shell se llama ruta. La ruta contiene muchos directorios que se usan comúnmente para almacenar comandos. Si un comando no está en la ruta, el usuario debe especificar su ubicación, o el shell no podrá encontrarlo. Los usuarios pueden agregar fácilmente directorios a la ruta, si es necesario.
Para invocar un comando a través del shell, simplemente escriba su nombre. El shell intentará encontrarlo en la ruta del sistema y ejecutarlo.

Trabajar con archivos de texto

Linux tiene editores de texto diferentes, con varias características y funciones. Algunos editores de texto incluyen interfaces gráficas, mientras que otros son herramientas solo de línea de comandos. Cada editor de texto incluye un conjunto de funciones diseñado para admitir un tipo específico de tarea. Algunos editores de texto se centran en el programador e incluyen funciones como resalto de sintaxis, comprobación de corchetes y paréntesis, y otras funciones centradas en la programación.
Si bien los editores de texto gráficos son convenientes y fáciles de usar, los editores de texto basados en líneas de comandos son muy importantes para los usuarios de Linux. El principal beneficio de los editores de texto basados en línea de comandos es que permiten la edición de archivos de texto desde una computadora remota.
Considere el siguiente escenario: un usuario debe realizar tareas administrativas en una computadora Linux pero no está sentado frente a ese equipo. Usando SSH, el usuario inicia una shell remoto a la computadora remota. bajo el shell remotor basado en texto, la interfaz gráfica no está disponible, lo que hace que sea imposible confiar en herramientas como los editores de texto gráficos. En este tipo de situaciones, los programas basados en texto son cruciales.
Debido a la falta de soporte gráfico, nano ( o GNU nano) solo se puede controlar con el teclado. Por ejemplo, CTRL + O guarda el archivo actual, CTRL + W abre el menú de búsqueda. GNU nano usa una barra de atajos de dos líneas en la parte inferior de la pantalla, donde se enumeran los comandos para el contexto actual. Presionando CTRL + G para la pantalla de ayuda y una lista completa de comandos.

La importancia de los archivos de texto en Linux

En Linux, todo se trata como un archivo. Esto incluye la memoria, los discos, el monitor y los directorios. Por ejemplo, desde el punto de vista del sistema operativo, mostrar información en la pantalla significa escribir en el archivo que representa el dispositivo de pantalla. No debería sorprender que la propia computadora esté configurada a través de archivos. Conocidos como archivos de configuración, generalmente son archivos de texto que se utilizan para almacenar ajustes y configuraciones para aplicaciones o servicios específicos. Prácticamente todo en Linux se basa en archivos de configuración para funcionar. Algunos servicios no tienen uno, sino varios archivos de configuración.
Los usuarios con los niveles de permiso adecuados pueden utilizar editores de texto para cambiar el contenido de los archivos de configuración. Una vez realizados los cambios, el archivo se guarda y puede ser utilizado por el servicio o la aplicación relacionados. Los usuarios pueden especificar exactamente cómo quieren que se comporte cualquier aplicación o servicio. Cuando se inician, los servicios y las aplicaciones verifican el contenido de archivos de configuración específicos para ajustar su comportamiento en consecuencia.
En la figura, el administrador abrió el archivo de configuración del host en nano para editarlo. El archivo de host contiene asignaciones estáticas de direcciones IP de host a nombres. Los nombres sirven como atajos que permiten conectarse a otros dispositivos usando un nombre en lugar de una dirección IP. Solo el superusuario puede cambiar el archivo de host. Nota: El administrador usó el comando sudo nano / etc / hosts para abrir el archivo. El comando sudo (abreviatura de "superuser do") invoca el privilegio de superusuario para usar el editor de texto nano para abrir el archivo host.

Clientes y servidores Linux

Introducción a las comunicaciones cliente-servidor.

Los servidores son computadoras con software instalado que les permite brindar servicios a los clientes a través de la red. Existen muchos tipos de servicios. Algunos proporcionan recursos externos como archivos, mensajes de correo electrónico o páginas web a los clientes que lo soliciten. Otros servicios ejecutan tareas de mantenimiento como administración de registros, administración de memoria, escaneo de disco y más. Cada servicio requiere un software de servidor independiente. Por ejemplo, el servidor de la figura utiliza software de servidor de archivos para brindar a los clientes la capacidad de recuperar y enviar archivos. Servidores, servicios y sus puertos

Para que una computadora pueda ser el servidor de múltiples servicios, se utilizan puertos. Un puerto es un recurso de red reservado utilizado por un servicio. Se dice que un servidor está "escuchando" en un puerto cuando se ha asociado a ese puerto.
Si bien el administrador puede decidir qué puerto usar con cualquier servicio dado, muchos clientes están configurados para usar un puerto específico de forma predeterminada. Es una práctica común dejar el servicio ejecutándose en su puerto predeterminado. La tabla enumera algunos puertos de uso común y sus servicios. Estos también se denominan "puertos conocidos".

Los navegadores web son clientes web que se utilizan para comunicarse con servidores web a través del Protocolo de transferencia de hipertexto (HTTP) en el puerto 80. Hardening devices

El endurecimiento del dispositivo implica implementar métodos probados para asegurar el dispositivo y proteger su acceso administrativo. Algunos de estos métodos implican mantener contraseñas, configurar funciones mejoradas de inicio de sesión remoto e implementar un inicio de sesión seguro con SSH. La definición de roles administrativos en términos de acceso es otro aspecto importante de la protección de los dispositivos de infraestructura porque no todo el personal de tecnología de la información debe tener el mismo nivel de acceso a los dispositivos de infraestructura. Dependiendo de la distribución de Linux, muchos servicios están habilitados de forma predeterminada. Algunas de estas funciones están habilitadas por motivos históricos, pero ya no son necesarias. Detener dichos servicios y asegurarse de que no se inicien automáticamente en el momento del arranque es otra técnica de refuerzo del dispositivo. Las actualizaciones del sistema operativo también son extremadamente importantes para mantener un dispositivo reforzado.
Todos los días se descubren nuevas vulnerabilidades. Los desarrolladores de sistemas operativos crean y publican correcciones y parches con regularidad. Es menos probable que una computadora actualizada se vea comprometida. Las siguientes son las mejores prácticas básicas para el refuerzo de dispositivos.

  • Garantizar la seguridad física.
  • Minimizar los paquetes instalados.
  • Deshabilitar los servicios no utilizados.
  • Use SSH y desactive el inicio de sesión de la cuenta raíz a través de SSH.
  • Mantenga el sistema actualizado.
  • Deshabilitar la detección automática de USB.
  • Hacer cumplir contraseñas seguras.
  • Forzar cambios periódicos de contraseña.
  • Evite que los usuarios reutilicen contraseñas antiguas.
Existen muchos otros pasos y, a menudo, dependen del servicio o de la aplicación.

Monitoreo de registros de servicio.

Los archivos de registro son los registros que almacena una computadora para realizar un seguimiento de los eventos importantes. Los eventos de kernel, servicios y aplicaciones se registran en archivos de registro. Es muy importante que un administrador revise periódicamente los registros de una computadora para mantenerla saludable. Al monitorear los archivos de registro de Linux, un administrador obtiene una imagen clara del rendimiento de la computadora, el estado de seguridad y cualquier problema subyacente. El análisis de archivos de registro permite al administrador protegerse de los problemas futuros antes de que ocurran.
En Linux, los archivos de registro se pueden clasificar como:

  • Registros de aplicaciones.
  • Registros de eventos.
  • Registros de servicio.
  • Registros del sistema.
Algunos registros contienen información sobre los demonios que se ejecutan en el sistema Linux. Un demonio es un proceso en segundo plano que se ejecuta sin la necesidad de interacción del usuario. Por ejemplo, System Security Services Daemon (SSSD) administra el acceso remoto y la autenticación para las capacidades de inicio de sesión único. Los tipos de sistema de archivos en Linux

Hay muchos tipos diferentes de sistemas de archivos, que varían en propiedades de velocidad, flexibilidad, seguridad, tamaño, estructura, lógica y más. Depende del administrador decidir qué tipo de sistema de archivos se adapta mejor al sistema operativo y los archivos que almacenará. La tabla enumera algunos tipos de sistemas de archivos que se encuentran comúnmente y son compatibles con Linux.

Sistema de archivos Linux. Descripción
ext2 ( segundo sistema de archivos extendido) ext2 era el sistema de archivos predeterminado en varias distribuciones importantes de Linux hasta que fue reemplazado por ext3.
Casi totalmente compatible con ext2, ext3 también admite el registro en diario (ver más abajo).
ext2 sigue siendo el sistema de archivos de elección para los medios de almacenamiento basados ​​en flash porque la falta de un diario aumenta el rendimiento y minimiza el número de escrituras.
Debido a que los dispositivos de memoria flash tienen un número limitado de operaciones de escritura, minimizar las operaciones de escritura aumenta la vida útil del dispositivo.
Sin embargo, los kernels contemporáneos de Linux también son compatibles con ext4, un sistema de archivos aún más moderno, con mejor rendimiento y que también puede funcionar en un modo sin diario.
ext3 ( tercer sistema de archivos extendido) ext3 es un sistema de archivos con diario diseñado para mejorar el sistema de archivos ext2 existente.
Un diario, la característica principal agregada a ext3, es una técnica utilizada para minimizar el riesgo de corrupción del sistema de archivos en caso de una pérdida repentina de energía.
Los sistemas de archivos mantienen un registro (o diario) de todos los cambios del sistema de archivos que se van a realizar.
Si la computadora falla antes de que se complete el cambio, el diario puede usarse para restaurar o corregir cualquier problema eventual creado por la falla.
El tamaño máximo de archivo en los sistemas de archivos ext3 es de 32 TB.
ext4 ( cuarto sistema de archivos extendido) Diseñado como sucesor de ext3, ext4 se creó en base a una serie de extensiones de ext3.
Si bien las extensiones mejoran el rendimiento de ext3 y aumentan el tamaño de los archivos admitidos, los desarrolladores del kernel de Linux estaban preocupados por los problemas de estabilidad y se oponían a agregar las extensiones al ext3 estable.
El proyecto ext3 se dividió en dos; uno se mantuvo como ext3 y su desarrollo normal y el otro, llamado ext4, incorporó las extensiones mencionadas.
NTFS ( sistema de archivos de red) NFS es un sistema de archivos basado en red que permite el acceso a archivos a través de la red. Desde el punto de vista del usuario, no hay diferencia entre acceder a un archivo almacenado localmente o en otra computadora en la red. NFS es un estándar abierto que permite que cualquiera lo implemente.
CDFS ( sistema de archivos de disco compacto) CDFS se creó específicamente para discos ópticos.
Sistema de archivos de intercambio Linux utiliza el sistema de archivos de intercambio cuando se queda sin RAM. Técnicamente, es una partición de intercambio que no tiene un sistema de archivos específico, pero es relevante para la discusión del sistema de archivos. Cuando esto sucede, el kernel mueve el contenido de RAM inactivo a la partición de intercambio en el disco. Si bien las particiones de intercambio (también conocidas como espacio de intercambio) pueden ser útiles para las computadoras Linux con una cantidad limitada de memoria, no deben considerarse como una solución primaria. La partición de intercambio se almacena en un disco que tiene velocidades de acceso mucho más bajas que la RAM.
HFS Plus o HFS+ (Sistema de archivos jerárquico Plus) Un sistema de archivos utilizado por Apple en sus computadoras Macintosh. El kernel de Linux incluye un módulo para montar HFS + para operaciones de lectura y escritura.
APFS ( sistema de archivos de Apple) Un sistema de archivos actualizado que utilizan los dispositivos Apple. Proporciona un cifrado sólido y está optimizado para unidades flash y de estado sólido.
MBR (Regristro de arranque maestro) Ubicado en el primer sector de una computadora particionada, el MBR almacena toda la información sobre la forma en que está organizado el sistema de archivos. El MBR cede rápidamente el control a una función de carga, que carga el sistema operativo.
Montaje es el término utilizado para el proceso de asignar un directorio a una partición. Después de una operación de montaje exitosa, se puede acceder al sistema de archivos contenido en la partición a través del directorio especificado. En este contexto, el directorio se denomina punto de montaje para ese sistema de archivos. Los usuarios de Windows pueden estar familiarizados con un concepto similar; la letra de la unidad. El resultado del comando muestra el resultado del comando mount emitido en Cisco CyberOPS VM.
Cuando se emite sin opciones, mount devuelve la lista de sistemas de archivos montados actualmente en una computadora Linux. Si bien muchos de los sistemas de archivos que se muestran están fuera del alcance de este curso, observe el sistema de archivos raíz (resaltado). El sistema de archivos raíz está representado por el símbolo "/" y contiene todos los archivos en la computadora de forma predeterminada. También se muestra en la salida que el sistema de archivos raíz se formateó como ext4 y ocupa la primera partición de la primera unidad (/dev/sda1).
